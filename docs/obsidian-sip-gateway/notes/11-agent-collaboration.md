# AI / Agent 协作规则

来源页面：`http://127.0.0.1:9100/docs/agent-readme`

这篇笔记用于沉淀当前项目的 AI / Agent 协作方式。长期规则应优先同步到仓库根目录的 `AGENTS.md`，因为它是 Agent 实际执行时读取的项目规则。

## 1. 沟通方式

- 全程使用中文回复。
- 每次回复都称呼用户为「利哥」。
- 尊重事实高于迎合用户。如果用户判断有误，需要直接指出，并说明原因。
- 不要只给结论，要说明判断依据、边界和不确定点。
- 如果需求不清楚，必须先问问题；没有问题后再执行。
- 用户要求“有问题直接问我”时，不能擅自假设高风险业务参数。

## 2. 分析方式

- 必须从第一性原理分析问题：先拆清楚链路、边界、输入输出、责任组件。
- 分析时区分：已实现、已验证、只是理论可行、当前未实现、商用前仍需验证。
- 不要把“本地测试通过”说成“真实商用可用”。
- 对 SIP / 电话链路问题，必须区分本地软电话测试、FreeSWITCH 本地容器、真实 SIP trunk、RTP / PCMA / NAT / 防火墙、AI 实时媒体链路。
- 对延迟、卡顿、断音、尾字丢失、打断异常等问题，必须拆成 SIP 信令、RTP / 音频帧、VAD、模型响应、下行播放队列、FreeSWITCH 播放完成事件、Media Adapter 等具体环节。
- 解决问题时优先找根因，不优先做表层补丁。
- 当用户强调“商用系统”时，必须按业内最佳实践分析，不能只给临时可用方案。
- 涉及第三方服务、模型、SDK、SIP trunk、软电话工具等外部信息时，需要优先查官方文档或可靠来源。

## 3. 执行方式

- 按阶段推进，每个阶段都必须可测试。
- 每完成一个阶段后，先停下来，让用户验证测试结果；不要自动进入下一阶段。
- 每个阶段开始前，要说明本阶段目标、会改什么、怎么测试、通过标准和不包含什么。
- 如果测试失败，先分析日志和现象，再给根因判断。
- 如果存在多个修复方案，要说明每个方案的影响、风险、是否根因修复，再让用户选择。
- 不要为了“看起来有进展”做无意义修改。
- 如果当前阶段的修改会被下一阶段废弃，要提前说明，不要做浪费性工作。

## 4. 文档要求

- 文档必须适合交接给同事，不能只适合当前电脑或当前会话。
- 文档要明确完整业务链路、当前实现、当前验证、剩余未实现、本地测试链路和真实商用链路的区别、数据格式、配置项、注意事项、测试方法、失败排查方式和最终商用形态。
- 文档里要使用图示描述链路，优先用 Mermaid。
- 不要把本机 IP、本机路径写成同事必须一致的前提。
- 密钥、token、secret 不能写入文档或提交到仓库。
- 已废弃路线要清理，避免同事误以为还有多条运行时主线。
- 当前路线、后续路线、最终商用路线要分开写清楚。

## 5. Git 要求

- 不要在用户未验证前提交。
- 不要在用户没有明确要求时提交。
- 提交前必须先说明本次 commit 准备包含什么，让用户确认。
- 用户确认后再提交。
- 后续修改如果用户要求新分支，必须在新分支继续。
- 不要回滚用户手动改动，除非用户明确要求。
- Commit 内容要简洁，描述当前验证了什么、属于整体链路哪一部分。
- 用户偏好中文说明 commit 内容；实际 commit message 仍需尊重项目约定。

## 6. 当前项目特别要求

- 当前主线是独立的 `sip-realtime-voice-gateway`，不是旧 TEN 级联链路。
- 当前 AI 主链路已收敛到豆包 S2S 端到端实时语音。
- 阿里 Realtime、火山硬件智能体、旧 ASR/LLM/TTS 级联链路不再作为运行时回退。
- 当前本地链路是：测试页 / HTTP `/calls`、FreeSWITCH originate、本地软电话 1000、9199、mod_audio_stream、实时媒体网关、豆包 S2S。
- 本地软电话只是测试替身：Windows 可用 MicroSIP，macOS 可用 Linphone / Zoiper，真实商用要替换为 SIP trunk。
- 当前 `mod_audio_stream` 可以继续用于验证，但商用前必须压测播放完成事件、打断、尾音、并发稳定性。
- 真实 SIP trunk 单路预验证可以先于业务系统回调和并发调度。
- 业务系统接入、持久化、webhook、并发限流、外呼队列可以后置，但上线前必须完成。

## 一句话原则

```text
先基于事实拆清楚链路和根因，按阶段做可验证实现，测试通过再继续；
不能把本地可用包装成商用可用。
```

## 和当前仓库规则的关系

这篇笔记是阅读和交接用的说明。Agent 执行时，以仓库根目录的 `AGENTS.md` 为准。

当前仓库 `AGENTS.md` 已明确：

```text
当前仓库根目录就是独立 SIP 实时语音网关项目。
不要再假设外层还有项目父目录，也不要引用旧 TEN 框架目录。
```
